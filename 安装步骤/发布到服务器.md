# 猫咪血统证书 — 发布到服务器（PM2 管理器）

目标域名：`https://cat.meituyin.cn`  
网站目录示例：`/www/wwwroot/cat.meituyin.cn`

> 本文假定站点、Nginx、SSL、PM2 管理器已可用。项目由 **PM2 跑 Node API（3001）**，Nginx 反代到本机 3001。不需要 PHP / MySQL。

---

## 1. 本地打包

在项目根目录（Windows PowerShell）：

```powershell
cd h:\Cat3
npm run build
```

压缩上传（**不要**包含 `node_modules`）：

| 必须带上 | 说明 |
|----------|------|
| `server/`、`dist/`、`package.json`、`package-lock.json` | 运行与前端 |
| `data/cat.db` | 迁移旧数据时必带 |
| `data/uploads/` | 用过图片列时带上 |
| `data/.installed` | 从本机迁移且已安装过时带上 |
| `.env` | 上传后在服务器改域名（见下） |
| `public/`、`font/` | 静态资源 |

---

## 2. 上传解压

宝塔 **文件** → 进入网站根目录 → 上传 zip → **解压**。  
解压后根目录应直接看到 `package.json`、`server/`、`dist/`。

**终端**（宝塔 SSH 或「终端」）执行一次：

```bash
cd /www/wwwroot/cat.meituyin.cn
chown -R www:www .
sudo -u www npm install
sudo -u www npm run build
```

> 不要用 Windows 拷过来的 `node_modules`；必须在 Linux 上 `npm install`。

---

## 3. PM2 管理器添加项目（核心）

打开宝塔 **PM2 管理器** → **添加项目**，填写：

| 配置项 | 填写 |
|--------|------|
| 项目名称 | `cat-svg` |
| 启动文件 | `server/index.js` |
| 运行目录 | `/www/wwwroot/cat.meituyin.cn` |
| 运行用户 | `www` |
| 端口 | `3001` |

保存后点击 **启动**。  
日志里应出现类似：`[CAT API] http://localhost:3001`。

日常在 PM2 管理器里：**重启 / 停止 / 查看日志**，不必记命令行。

---

## 4. 首次配置（二选一）

### A. 从本机迁移（已有 `data/cat.db`）

1. 已上传 `data/cat.db`、`data/.installed`、`.env`
2. 编辑网站根目录 `.env`：

```env
PORT=3001
JWT_SECRET=随机长字符串
ADMIN_USERNAME=admin
ADMIN_PASSWORD=你的强密码
CORS_ORIGIN=https://cat.meituyin.cn
NODE_ENV=production
```

3. PM2 管理器 → **重启** `cat-svg`  
4. **不要**再访问 `/install.html`（会重复初始化）

### B. 服务器全新安装（无数据库）

1. PM2 先 **启动** `cat-svg`
2. 浏览器打开：`https://cat.meituyin.cn/install.html`
3. 填写网站地址 `https://cat.meituyin.cn`、管理员账号、端口 `3001` → **开始安装**
4. 安装完成后 PM2 管理器 → **重启** `cat-svg`

---

## 5. Nginx 反代

网站 → **反向代理** → 目标 URL：

```text
http://127.0.0.1:3001
```

发送域名：`$host`。  
防火墙 / 安全组只开 **80、443**，**不要**对公网开放 3001。

---

## 7. 以后更新版本

1. 备份 `data/cat.db`、`.env`（及 `data/uploads/`）
2. 上传新代码覆盖（**保留** `data/`、`.env`）
3. 终端：

```bash
cd /www/wwwroot/cat.meituyin.cn
chown -R www:www .
sudo -u www npm install
sudo -u www npm run build
```

4. PM2 管理器 → **重启** `cat-svg`
5. 再打开 `/api/meta` 确认功能列表正常

---

## 8. 常见问题

| 现象 | 处理 |
|------|------|
| 502 | PM2 里 `cat-svg` 是否在线；反代是否指向 `127.0.0.1:3001` |
| 一直跳转 install | 确认存在 `data/.installed`；PM2 重启 |
| 登录失败 | 检查 `.env` 密码；或 SSH 执行 `npm run reset:admin` |
| 新功能 404 | 必须 `npm run build` + PM2 重启 |
| npm install 失败 | 服务器安装 gcc/g++ 后重试；Node 需 **20.19+** |
